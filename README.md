# 異形オセロ評価値計算 (corner-cut-reversi-eval)

複数の**異形オセロ盤面**で、各局面の**全合法手それぞれの評価値**を盤面に表示する
「検討盤(研究用)」アプリです。AI 対戦はしません。黒も白も自分でクリックして進め、
最善手・善手・悪手を色分けで確認しながら局面を研究できます。

**対応盤面(4種):**

| 盤面 | マス数 | 形 |
|------|-------|----|
| 通常 | 64 | 8×8、欠けなし |
| クロス | 48 | 四隅の 2×2 を除く |
| 八角 | 52 | 四隅を各 3 マス斜めにカット(八角形) |
| ホロー | 60 | B2・G2・B7・G7 の 4 マスを除く |

フロント完結(サーバ不要)・TypeScript + Vite。評価計算は Web Worker で
バックグラウンド実行し、UI を固めません。

## 起動方法

```bash
npm install
npm run dev
```

表示された `http://localhost:5173/`(空いていなければ別ポート)をブラウザで開きます。

本番ビルド:

```bash
npm run build       # 型チェック(tsc) + Vite ビルド → dist/
npm run preview     # dist/ をローカル配信して確認
```

> **Node のバージョン:** Vite 5 は **Node 18 以上**が必要です。
> 古い Node が有効な環境では `node -v` を確認し、18+ に切り替えてください。

エンジンの自動検証(盤面ロジック・探索の妥当性チェック):

```bash
npm run check:engine
```

## 操作説明

1. **盤面・先手の選択** — 起動すると検討盤が表示されます。上部ツールバーで「盤面の種類
   (通常 / クロス / 八角 / ホロー)」と「先手の色(黒 / 白)」をいつでも切り替えられます。
   切り替えると、その設定で局面を最初からやり直します。
2. **合法手をクリックして進める** — 評価値が表示された合法手マスをクリックすると 1 手進み、
   手番が交代して次局面の評価値を再計算・再表示します。非合法マスはクリックできません。
3. **評価値の見方** — 各合法手マスに数値(石差ベース、例 `+6` / `-4` / `±0`)が出ます。
   - **最善手** … アンバー(琥珀)の太い二重リングで最も目立たせます(その局面で一番良い手)。
   - **善手** … ティールグリーンの細リング(最善との差が小さい)。
   - **悪手** … ダルレッドの細リング(最善から大きく離れる)。
   - 色だけでなくリングの太さ・二重構造でも階層が分かります(最善 > 善手 > 悪手)。
4. **補助機能**
   - **1手戻る(待った)** — 直前の局面に戻ります。
   - **リセット** — 現在の盤面・先手設定のまま、局面を最初からやり直します。
   - **パス自動処理** — 打てる手がない手番は自動でパスし、画面下に通知します。
   - **終局表示** — 両者とも打てなくなったら、最終石数と勝敗を表示します。

> 評価エンジンは終盤(空き16以下)で「完全読み」モードになり、**確定の最終石差**を評価値にします。

## 異形ルールの説明

通常オセロ(8×8、8方向の挟み反転)に準拠しますが、以下が異なります。

- **盤面は4種**(通常64 / クロス48 / 八角52 / ホロー60)。それぞれ「欠けマス」が異なります。
  欠けマス座標(row 0–7 = 上→下、col 0–7 = 左→右):
  - **クロス**: 四隅の 2×2 — 行0-1×列0-1 / 行0-1×列6-7 / 行6-7×列0-1 / 行6-7×列6-7(計16)
  - **八角**: 四隅を各3マス — 左上(0,0)(0,1)(1,0) / 右上(0,6)(0,7)(1,7) / 左下(6,0)(7,0)(7,1) / 右下(6,7)(7,6)(7,7)(計12)
  - **ホロー**: (1,1)(1,6)(6,1)(6,6) = B2・G2・B7・G7(計4)
  - **通常**: 欠けマスなし
- **欠けマスには石を置けず、反転にも一切関与しません。** 挟みのライン上では
  盤外と同じ「壁」として扱われます(欠けマスをまたいで挟むことはできない)。
- **初期配置は中央4マス**(全盤面共通、中央は欠けません)。黒=(3,3)(4,4) / 白=(3,4)(4,3)。
- 合法手 = 1つ以上の相手石を反転できる手。置ける手がなければパス。両者パスで終局。
- **終局スコアは盤上の石数そのまま**です。盤が埋まりきらずに終局しても、
  空きマスを勝者に加算しません(オセロ公式の空きマス加算ルールは採用していません)。

> 盤面によって四隅の形が変わるため、通常オセロの「角を取れば安定」という定石は
> そのままでは使えません。評価関数は異形盤向けに調整した暫定版です(下記「既知の制約」参照)。

## 評価エンジンの概要

- **アルゴリズム:** ネガマックス + αβ枝刈り + move ordering + 置換表 + **反復深化**。
- **思考時間制:** 中盤は 1 手あたり上限 **2秒**、終盤は **3秒**(定数で管理)。
  反復深化で時間内に到達できた最深の結果を使います。時間切れになっても UI は必ず復帰します。
- **終盤完全読み:** 空きマスが **16以下**で最後まで読み切り、確定の最終石差を評価値にします。
- **盤面サイズ非依存:** マス重みは盤面の形から手続き生成し、置換表・完全読みも全4盤面で機能します。
- **各手の評価値** = その手を打った後の局面を相手番から探索した値の符号反転。

時間制・完全読みライン・各種しきい値は `src/engine/search.ts` の冒頭定数で調整できます。

## 既知の制約・未調整箇所

- **評価関数の精度(中盤):** マス重み等は暫定で、盤面によって精度に差があります
  (通常盤が最良、八角・ホローは角理論が通常と異なるため粗め)。明らかな好手/悪手は
  区別できますが、絶対値の精度は粗めです。確定石・開放度・パリティ等の項は未実装。
- **中盤探索のスコア尺度:** 終局葉(確定石差)と中盤ヒューリスティック葉のスケールが
  一部混在しており、稀に「確実な勝ち」を過小評価する可能性があります(勝敗の符号は常に正しく、
  終盤の完全読み区間は正確)。将来のチューニング候補です。
- **色分けのしきい値:** 善手/悪手の境界(`src/ui/evalColor.ts`)は仮置きです。実測して調整する想定。
- **中盤の数値は目安:** 確定値は終盤完全読み区間(空き16以下)のみです。
- **性能は環境依存:** 本環境の確認では中盤で 1 手あたり 1〜2 秒程度。低速環境では時間上限に
  張り付き到達深さが下がります。重い場合は終盤完全読みの開始ライン(残り16)を下げてください。
```
