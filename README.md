# 異形オセロ評価値計算

四隅の形が異なる複数の盤面で、各局面の打てる手すべてに評価値を表示する、リバーシ(オセロ)の検討盤 Web アプリ。

---

## 特徴

- **形の異なる複数の変則盤面を切り替えて研究** — 通常の盤に加え、四隅を削った変則盤を、同じ操作感のまま切り替えて検討できる
- **打てる手すべてに評価値を表示** — その局面で着手できるマス全部に評価を出し、最善手・善手・悪手を色で直感的に示す
- **一画面で完結する操作** — 盤面の種類と先手の色をいつでも切り替えられ、切り替えると即その設定で並び直す
- **重い思考中も操作が固まらない** — 評価の計算を画面の処理から切り離しているため、考えている間も UI が固まらない
- **終盤は読み切って確定評価** — 終盤に入ると最後まで読み切り、確定した結果を評価値として示す

---

## 技術スタック

| レイヤー | 採用技術 |
|---|---|
| 言語 | TypeScript(strict) |
| ビルド / 開発 | Vite |
| UI | 素の DOM(UI フレームワークなし) |
| 並行処理 | Web Worker |
| テスト | Node 実行のエンジン検証スクリプト |
| 配信 | 静的サイト(サーバー不要) |

### 実装上の工夫

- **盤面をデータとして定義し、ロジックを盤面非依存に** — 欠けマスの集合だけで盤面を表現し、合法手生成・反転・探索を形に依存しない作りにした。盤面の追加や自由編集にも拡張しやすい。
- **評価計算を Web Worker に分離** — 探索を別スレッドへ逃がして画面をブロックしない。思考時間を超過した場合も暫定結果へ切り替え、操作不能にならないよう多重に保護している(`src/worker/`, `src/engine/search.ts`)。
- **レイアウトのちらつき対策** — 盤面グリッドのサイズを固定し、評価値は重ね描きにすることで、手が進んでもマスの寸法が変わらないようにした。
- **可読性を優先した評価表示** — 暗い盤面でも数字が沈まないよう、色とリング形状の二重符号化で最善・善手・悪手を区別した(色覚特性にも配慮)。
- **探索の構成** — 反復深化と αβ 枝刈り、置換表を組み合わせ、終盤は完全読みに切り替える(`src/engine/`)。

## ローカル起動方法

```bash
# 1. 依存をインストール
npm install

# 2. 開発サーバー起動
npm run dev
# → http://localhost:5173
```

起動するとすぐ検討盤が表示され、盤面の種類と先手の色を選んで局面を進められる。サーバーや API キーは不要で、すべてブラウザ内で動作する。

> Node.js は 18 以上を推奨(Vite 5 の要件)。

## ディレクトリ構成(抜粋)

```
src/
├ engine/         # 盤面ロジックと探索(純粋ロジック)
│  ├ types.ts     # 盤面表現・盤面プリセット定義
│  ├ board.ts     # 合法手生成 / 反転 / パス・終局判定
│  ├ evaluate.ts  # 中盤の評価関数
│  ├ zobrist.ts   # 置換表キー
│  └ search.ts    # 探索(反復深化 + αβ + 終盤完全読み)
├ worker/         # 評価計算をバックグラウンド実行する Worker
├ game/           # ゲーム進行(着手 / 待った / 終局)
└ ui/             # 画面描画・操作・Worker 連携
scripts/
└ engine-check.ts # エンジンの自動検証
```

---

## データ方針

サーバーを持たず、すべての処理がブラウザ内で完結する。対局内容や操作を外部へ送信・保存することはない。
